# /findata-classify

根据《金融信息服务数据分类分级指南》附录 A（67 类三级分类）对金融数据分类分级。

## 用法

```
/findata-classify 身份证号
/findata-classify 公司员工姓名
/findata-classify 股票实时行情
```

## 依赖

```bash
pip install pymupdf
```

本技能使用 PyMuPDF (`fitz`) 读取 PDF 中的附录 A 表格。`requirements.txt` 位于技能目录中。

## 参考文件

- `金融信息服务数据分类分级指南.pdf` — 正式稿原文
- `knowledge/authoritative.json` — 从 PDF 附录 A 自动提取的 67 条分类记录
- `knowledge/custom.json` — 用户自定义覆盖（优先级最高）

## 执行步骤

### Step 1 — 关键字查找

```bash
classifier.py [-o output.json] <query>
```

`classifier.py` 执行三层关键字匹配，返回最多 20 个侯选：

| 优先级 | 来源 | 说明 |
|--------|------|------|
| 最高 | `custom.json` | 用户自定义，覆盖权威数据 |
| 中 | `authoritative.json` | 从 PDF 附录 A 自动提取的 67 条记录 |
| 最低 | 原始 PDF | 关键字未命中时，大模型用 PyMuPDF `find_tables()` 直接读取 |

每行包含：一级分类、二级分类、三级分类、描述和示例、参考最低级别和升级条件。

**注意**：`classifier.py -o file.json query` 将结果写入文件，避免控制台编码问题。

### Step 2 — 语义匹配（大模型）

- **有侯选**：对比各侯选的描述和上下文，用语义理解判定最佳匹配，赋值语义匹配度（0-100）
- **无侯选**：用 PyMuPDF 重新读取 PDF 中附录 A 的表格，逐行扫描描述和三级分类名，用语义理解找到最匹配的行，然后构造结果

### Step 3 — 输出最终 JSON

## 输出格式

```json
{
  "数据名称": "身份证号",
  "语义匹配度": 90,
  "分类": { "一级": "用户数据", "二级": "个人用户数据", "三级": "个人用户基本信息" },
  "分级": "敏感一般数据",
  "升级条件": ["当1000万人及以上的个人用户基本信息数据集 → 重要数据"],
  "理由": "依据《金融信息服务数据分类分级指南》附录A，身份证号属于「个人用户基本信息」中「证件号码」的典型示例，参考最低级别为「敏感一般数据」。涉及1000万人及以上数据集时升级为「重要数据」。"
}
```

## 语义匹配度说明

| 分值 | 含义 |
|------|------|
| 95-100 | 查询词与该分类条目的语义完全一致 |
| 80-94 | 强相关，是该分类的典型子集或同义表达 |
| 60-79 | 相关，但存在一定歧义 |
| <60 | 弱相关，仅在关键字层面命中但语义偏离 |

## 分级体系（四级）

| 级别 | 说明 |
|------|------|
| 核心数据 | 关系国家安全重点领域 |
| 重要数据 | 特定领域/群体/区域，一定精度和规模 |
| 敏感一般数据 | 对经济运行/社会秩序/公共利益有一定影响，或对组织/个人造成重要影响 |
| 常规一般数据 | 其他 |

## 分级解析规则

`parse_grade()` 自动从「参考级别和条件」字段解析默认级别和升级条件：

- **默认级别** = 文本中最后出现的级别（通常是"其他"兜底情况）
- **升级条件** = 非默认级别及其对应的前置条件
- PDF 提取的产物（如"重要数 据"中间空格）会被自动归一化

## 规模阈值（升级为重要数据）

| 数据类型 | 阈值 |
|----------|------|
| 个人用户基本信息（含身份证号） | ≥1000万人 |
| 个人用户交易数据 | ≥100万人 |
| 生物特征识别信息 | ≥10万人 |
| 机构用户基本信息 | ≥100万机构 |
| 机构用户交易数据 | ≥10万机构 |
| 行业指标数据（精度高于官方且反映省域以上范围） | 精度/时效/维度条件 |
